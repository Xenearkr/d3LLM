# scripts

## merge_dream_lora.py：合并 Base + LoRA checkpoint

将 `d3llm_dream_train.py` 训练得到的 LoRA checkpoint 与 base 模型合并为完整权重并保存，便于：

- 使用 `eval_scripts/run_code_eval.sh`、`dream_*.sh`、`measure_tpf_tps.py` 等直接指定本地路径评估
- 使用 `chat/chat_d3llm_dream.py` 时加载本地合并后的目录
- 上传到 HuggingFace 或在其他环境仅依赖 transformers 加载（无需 PEFT）

### 依赖

- 在仓库根目录执行：`cd /path/to/d3LLM`
- 已安装：`transformers`, `peft`, `torch`

### 用法

**方法1. 使用默认路径（当前仓库最新一次 DREAM 训练 + merge_dream_lora 中指定的 checkpoint）**

- 在scripts/merge_dream_lora.py中修改main函数的checkpoint_dir参数为需要合并的checkpoint目录
- 在d3llm根目录运行：
```bash
python scripts/merge_dream_lora.py
```

合并结果默认保存到：`output_model/d3LLM_DREAM_merged/`

**方法2. 指定 checkpoint 与输出目录（示例）**

```bash
python scripts/merge_dream_lora.py \
  --checkpoint_dir output_model/d3LLM_DREAM_local_0312_043946/checkpoint-5742 \
  --output_dir output_model/d3LLM_DREAM_merged
```

**方法3. 指定 base 模型（需与训练时一致）（示例）**

```bash
python scripts/merge_dream_lora.py \
  --base_model Dream-org/Dream-v0-Instruct-7B \
  --checkpoint_dir output_model/d3LLM_DREAM_local_0312_043946/checkpoint-5742 \
  --output_dir output_model/d3LLM_DREAM_merged
```

**4. 合并后评估示例**

- 测 TPF/TPS：
  ```bash
  python eval_scripts/measure_tpf_tps.py --model_type d3llm --model_name output_model/d3LLM_DREAM_merged --dataset humaneval
  ```
- 代码评测（HumanEval/MBPP）：
  ```bash
export D3LLM_DREAM_CHECKPOINT="output_model/d3LLM_DREAM_merged"
export HF_ALLOW_CODE_EVAL=1
bash eval_scripts/run_code_eval.sh d3llm_dream humaneval
bash eval_scripts/run_code_eval.sh d3llm_dream mbpp
```

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--base_model` | Dream-org/Dream-v0-Instruct-7B | 与训练时一致的 base 模型 |
| `--checkpoint_dir` | output_model/d3LLM_DREAM_local_0312_043946/checkpoint-5742 | LoRA 目录（含 adapter_config.json） |
| `--output_dir` | output_model/d3LLM_DREAM_merged | 合并后保存目录 |
| `--torch_dtype` | bfloat16 | 保存权重 dtype |

---

### 完整操作：合并 + 评估（从 checkpoint 到 HumanEval/MBPP）

在仓库根目录执行（路径按你本机实际 run 与 checkpoint 调整）：

```bash
# 1. 合并 base + LoRA（默认使用 output_model/d3LLM_DREAM_local_0312_043946/checkpoint-5742）
python scripts/merge_dream_lora.py \
  --checkpoint_dir output_model/d3LLM_DREAM_local_0312_043946/checkpoint-5742 \
  --output_dir output_model/d3LLM_DREAM_merged

# 2. 设置环境变量指向合并后的目录（相对仓库根即可）
export D3LLM_DREAM_CHECKPOINT="output_model/d3LLM_DREAM_merged"

# 3. 跑 HumanEval / MBPP（需先 export HF_ALLOW_CODE_EVAL=1）
export HF_ALLOW_CODE_EVAL=1
bash eval_scripts/run_code_eval.sh d3llm_dream humaneval
bash eval_scripts/run_code_eval.sh d3llm_dream mbpp
```

结果目录：`utils/utils_Dream/eval_instruct/eval_tmp/code_eval_d3llm_dream_humaneval/` 与 `code_eval_d3llm_dream_mbpp/`。
