# HumanEval / MBPP 评估说明

## 评估方法

在仓库根目录执行：

```bash
export HF_ALLOW_CODE_EVAL=1
bash eval_scripts/run_code_eval.sh <模型> <数据集>
```

- **模型**：`d3llm_dream` | `d3llm_llada` | `vanilla_dream` | `vanilla_llada`
- **数据集**：`humaneval` | `mbpp`

示例：

```bash
bash eval_scripts/run_code_eval.sh d3llm_dream humaneval
bash eval_scripts/run_code_eval.sh vanilla_llada mbpp
```

结果与日志：

- **Dream 系**：`utils/utils_Dream/eval_instruct/eval_tmp/code_eval_<模型>_<数据集>/`，内含 `results_*.json`、`samples_*_instruct_*.jsonl`
- **LLaDA 系**：`utils/utils_LLaDA/evals_results/<d3llm|GSAI-ML>/<数据集>-run_code_eval-<模型>-ns*-256/<模型名>/`，内含 `results_*.json`、**HumanEval** 为 `samples_humaneval_*.jsonl`（任务 **humaneval**），**MBPP** 为 `samples_mbpp_*.jsonl`。LLaDA 系 HumanEval 的 pass@1 以对上述 samples 运行 `postprocess_code_humaneval.py` 后的输出为准（lm_eval 表格中为 0 属预期）。
